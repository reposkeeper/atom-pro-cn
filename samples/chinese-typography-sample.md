# 中文排版样张

这是一段用于测试中文正文节奏的内容。好的主题不只是颜色好看，还要让中文段落在长时间阅读时保持稳定的灰度、足够的呼吸感，以及清楚但不过分张扬的层级。

当段落中混入 English words、`inline code`、数字 12345 和链接时，行距不应该突然显得拥挤。这个主题默认采用均衡排版，目标是在技术笔记和长文写作之间找到舒服的位置。

## 列表节奏

- 第一条列表项应该和正文保持同一套行高，但条目之间需要有一点停顿。
- 第二条列表项包含更长的中文说明，用来观察换行后的内部节奏是否稳定。多行列表项不应该像两个段落，也不应该紧贴成一块。
- 第三条列表项包含 `inline code`，测试代码片段是否破坏中文句子的视觉连续性。

1. 有序列表第一项用于测试数字标记。
2. 有序列表第二项用于测试中文内容和标号之间的距离。
3. 有序列表第三项继续观察行距和段距。

- 嵌套列表外层
  - 嵌套列表内层应该稍微收紧，避免层级越深空隙越大。
  - 内层第二项继续测试多行中文内容。这里故意写长一点，看看换行后是否仍然自然。
    - 第三层列表要能看清层级，但不应显得松散。

- [ ] 未完成任务，需要清晰但不抢眼。
- [x] 已完成任务，需要保持可读。

## 引用和提示

> 中文引用块需要左边界提供结构感，但正文仍然应该好读。引用内部的段落距可以略微收紧，避免短引用变得过高。
>
> 第二段引用用于检查引用块里的段落节奏。

> [!note] 一个普通提示
> Callout 的颜色应该提供语义，但不应该让中文正文被背景色压住。

> [!warning] 注意事项
> 警告块需要更醒目一些，但仍然要保持 Atom Pro CN 的冷静工作台气质。

## 代码

中文段落里的 `const theme = "Atom Pro CN"` 应该足够轻，不要像按钮一样跳出来。

```js
const palette = {
  dark: "#272b34",
  accent: "#61afef",
  cnLineHeight: 1.9,
};

function describeTheme(name) {
  return `${name} keeps Atom colors and improves Chinese typography.`;
}
```

### 三级标题

三级标题之后的段落用于观察中层标题的节奏。标题应该清楚，但不需要像网页落地页一样夸张。

#### 四级标题

四级标题适合笔记内部的小分节，字号和字重都应该克制。

##### 五级标题

五级标题和六级标题主要用于技术文档，不应破坏页面的整体灰度。
