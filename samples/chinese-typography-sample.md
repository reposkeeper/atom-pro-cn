# 中文排版样张

这是一段用于测试中文正文节奏的内容。好的主题不只是颜色好看，还要让中文段落在长时间阅读时保持稳定的灰度、足够的呼吸感，以及清楚但不过分张扬的层级。

当段落中混入 English words、`inline code`、数字 12345 和链接时，行距不应该突然显得拥挤。这个主题默认采用均衡排版，目标是在技术笔记和长文写作之间找到舒服的位置。

这一行用于测试下划线细节：<u>普通下划线应该和中文文字保持一点间隙</u>，链接在 hover 时也应该有类似 Notion 的轻微距离，而不是贴在字脚上。

## 长段落压力测试

这一段故意写得比较长，用来观察中文正文在一整块内容里形成的页面灰度是否稳定。很多主题在短句和标题附近看起来都不错，但一旦进入连续阅读，问题就会慢慢浮出来：行距太小会让汉字之间的横向密度和纵向密度叠在一起，整段文字像一块压实的灰色布料；段距太大又会把文章切得过碎，让思路像被不断打断。Atom Pro CN 的目标不是把每个段落都做成宽松的散文版式，而是在日常知识库里保持一种长期可用的中间态：会议记录、技术方案、读书摘录、项目复盘和临时想法都可以放进同一个视觉系统里，不需要因为内容类型变化而频繁切换主题或手动调字号。

第二个长段落用于测试中英文、数字和代码词汇混在中文句子里的表现。真实笔记往往不会完全是纯中文，比如我们会写到 API design、local-first、sync conflict、`metadataCache`、`theme.css`、版本号 `1.9.10`，也会在同一段里记录“这个方案暂时先上线，后面根据使用反馈再调”。如果内联代码的背景太重，中文句子就会被切成一段一段；如果英文字体和中文字体的比例不协调，行内节奏会突然变得起伏很大。这个样张希望能逼出这些细节：英文不应该显得像异物，数字不应该跳得太亮，内联代码应该被看见，但不应该像一个按钮那样抢走注意力。

第三个长段落更接近工作流笔记的语气，用来检查阅读视图和编辑视图之间是否一致。一个主题真正好不好用，常常不是在截图里决定的，而是在反复编辑、滚动、选择、折叠和搜索时决定的。中文用户写笔记时经常会一边整理上下文，一边把刚刚产生的判断追加到段落末尾；也会在一个段落里保留多个层次的语义，例如背景、限制、风险、下一步动作和暂时不处理的边界。如果主题的段落节奏不稳，用户会下意识地把内容拆成很多短行，表面上是为了清楚，实际上是主题没有承担好排版工作。这里保留较长的连续文字，就是为了让行高、段距、字重和颜色一起接受测试。

第四个长段落用于观察亮色模式和暗色模式的共同问题。暗色主题容易把对比做得太强，让浅色文字在深色背景上产生轻微眩光；亮色主题则容易把背景做得太白，让中文正文显得薄而散。Atom 的底色本来偏工程化，适合代码和结构化信息，但中文长文需要多一点温和的空间。我们希望暗色模式像一个安静的编辑器，亮色模式像一张不刺眼的工作纸：两者都应该允许用户连续阅读几屏内容，而不是只在第一眼显得有性格。真正的主题气质应该藏在这些细节里，越写越顺手，而不是越看越累。

## 列表节奏

- 第一条列表项应该和正文保持同一套行高，但条目之间需要有一点停顿。

- 第三条列表项包含 `inline code`，测试代码片段是否破坏中文句子的视觉连续性。

## 代码

中文段落里的 `const theme = "Atom Pro CN"` 应该足够轻，不要像按钮一样跳出来。

```js
const palette = {
  dark: "#272b34",
  accent: "#61afef",
  cnLineHeight: 1.9,
};

function describeTheme(name) {
  return `${name} keeps Atom colors and improves Chinese typography.`;
}
```
